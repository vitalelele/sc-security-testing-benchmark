# Smart Contract Security Testing Benchmark

This repository contains the experimental artifacts developed for the Master's thesis:

**"Beyond Fuzzing: A Comparative Study of Smart Contract Security Analysis Techniques"**  
Università degli Studi di Milano  
Supervisor: Prof. Andrea Lanzi  

---

## Overview

The goal of this project is to provide a **reproducible benchmark** for evaluating different smart contract security analysis techniques, focusing on:

- **Grey-box fuzzing** (Smartian)
- **Symbolic execution** (Mythril)
- **Snapshot-based fuzzing** (ItyFuzz)

The evaluation is performed on a curated dataset of vulnerable smart contracts, measuring:

- Detection capability (FOUND)
- Time-to-Discovery (TTD)
- Tool-specific behavior and limitations

#### Results Structure

The `results/` directory contains the outputs generated by each tool:

```

results/
├── ityfuzz/
├── mythril/
└── smartian/

```

Each tool follows a consistent structure:

- Organized by **contract**
- Each contract includes **5 independent runs** (`run1`–`run5`)

Example (ItyFuzz / Smartian):

```

<tool>/<contract>/
├── run1/
├── run2/
├── run3/
├── run4/
└── run5/

```

Each run contains execution logs and tool-specific artifacts (e.g., summaries, generated inputs, detected bugs).

Example (Mythril):

```

mythril/<category>/<contract>/
├── run1/
│   ├── stdout.log
│   ├── stderr.log
│   ├── report.json
│   ├── runtime_sec.txt
│   ├── status.txt
│   ├── json_status.txt
│   ├── exit_code.txt
│   └── meta.txt
├── run2/
├── run3/
├── run4/
└── run5/

```

---

### 2. SafeContract

Contains the **control dataset** (secure contracts), used to evaluate false positives.

```

SafeContract/
├── control_set/
├── scripts/
├── solutions/
└── results/

```

⚠️ **Important note**  
For the control dataset, full tool outputs are **not stored**.  
Results were obtained directly from console execution and manually extracted.

These results are fully reported in the aggregated datasets (see `GoogleSheet/`).

---

### 3. GoogleSheet

This directory contains the **processed datasets** extracted from raw tool outputs:

```

GoogleSheet/
├── ityfuzz_raw.csv
├── mythril_raw.csv
└── smartian_raw.csv

```

These files represent:

- Structured extraction of all runs
- Normalized data used for analysis in the thesis
- The primary source for tables, metrics, and comparisons

---

## Experimental Design

- **Dataset**: vulnerable + control contracts
- **Runs per contract**: 5
- **Execution mode**: sequential (no parallelization)
- **Environment**: WSL2 (Linux-based execution)

Each tool was evaluated under the same conditions to ensure comparability.

---

## Reproducibility

The repository includes:

- Raw execution outputs (`results/`)
- Execution scripts (`scripts/`)
- Aggregated datasets (`GoogleSheet/`)

This allows:

- Full inspection of tool behavior
- Reproduction of experiments
- Independent verification of results
